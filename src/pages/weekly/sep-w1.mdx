## 四、[信任的演化](https://audreyt.github.io/trust-zh-TW/)
博弈論告訴我們，要發展信任，需要掌握的三件事：
1. 重複的互動信任是保持人際關係長久的基石，但建立信任的前提，是知道彼此將會有重複的互動。
2. 「雙贏」並非不可能你必須進行非零和遊戲，這樣的博弈並不保證兩個玩家都會變得更好、達到雙贏，但至少必須要有達成雙贏的可能性。
3. 低機率的誤解如果誤解發生得過多，信任就會瓦解。但是，如果有一點點誤會的時候，它將使寬容蔓延開來。當然，現實世界裡的信任會受到更多因素的影響。比如，名譽、共同的價值觀、合約、文化傳統等等。不過，別忘了...

> 遊戲是什麼，直接決定了玩家做什麼。
我們今天的問題，不僅是人們正在失去信任，而是我們的環境正在違背信任演化的規律。

「我們僅僅是我們周遭環境的產物」，這個觀念看起來可能有點犬儒或者天真無邪，但是博弈論也在提醒著，我們就是彼此的環境啊。短期來講，遊戲決定玩家，但是長期來說，決定遊戲的正是我們這些玩家。

所以，我們每個人都行動起來，做你力所能及的，去創造一個可以讓信任演化的條件。建立人際關係、努力尋求雙贏、溝通儘量清晰。或許到了那個時候，我們能夠停止向對方開火，走出各自的戰壕，穿過戰爭的無人地帶來到彼此面前...

講解賽局理論，有很多關卡，後面能針對信任的獎懲、誤解度和人物設定都進行細微調整。
所以基本上應該可以玩出所有賽局理論的例子

## 五、「念念不忘，必有迴響」
最近開始認真刷 leetcode
以前看了好幾遍都看不懂的演算法或題目解法突然就看懂了
現在也開始很好奇以前到底在不懂什麼？明明就很直觀。
像是以前看到 Dynamic Programming 一直搞不懂什麼製表跟查表，現在就能明白最基本就是製作一個陣列來存值當成一個表，最後找答案的時候直接從陣列中找值這樣，可參考 Fibonacci Number 這題經典的 DP 作法。

不過認真要想為什麼的話，可能是以下幾點
1. 自己沒有先難倒自己
	- 以前自己總是會先覺得好難，一直用這種心態，簡單的題目都會被自己想像成很難。
2. 懂得分階段或是用不同的角度切入來理解概念
	- 以前太想一步登天，根本不懂得輪子的構造，就想開始造輪子
	- 我們學習知識是由已知去推導未知
	- 但是若我們的已知太少，推導未知就會變得非常困難
	- 所以假設我在對 DP 一無所知的情況下，直接去看 DP 相關的 Leetcode 題目，就算看了解答也會一頭霧水很久
	- 但如果我先對於 DP 的流程是「找規律、製表跟查表」有所理解，那我很快地就能把眼前的程式碼轉換成這個流程：哪幾行是在找規律、哪幾行在製表、最後怎麼查表。
	- 同樣的概念在我學會 Two Pointer 後，再回去看相關的題目也有同樣的感覺。
	- 為何這個變數要取名 left 跟 right？理解 Two Pointer 後馬上就能理解這邊是要用 left & right pointer 然後可能是走 sliding window 的方式找到解答。
3. 自己隨著時間變強了
	- 人生沒有白走的路，每一步都算數
	- 可能我無意間在哪些地方吸收的知識都促成了此刻我能看懂一些東西的條件
4. 有沒有遇到對的那篇文章
	- 知識的傳遞很奇妙，同樣的概念有時候換個說法就能點通
	- 而每個人能被點通的說法不太一樣
	- 有時候我們只是要找到那個可以讓我們理解的說法
	- 就算大家都說哪篇文章寫得好，medium 有幾千個拍手，但我就是讀不懂，然後在某個小網站上看到別人隨便記筆記的文章就通了。
	- 所以寫文章真的是懸壺濟世呀，你一定會幫到人的，只是你可能不知道
